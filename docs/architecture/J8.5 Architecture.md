# J8 Architecture

## Knowledge Platform Architecture

---

# Purpose

J8 introduces a persistent Knowledge Platform beneath the strategic reasoning engine.

The platform separates:

Knowledge Construction

from

Knowledge Consumption.

Knowledge becomes durable.

Reasoning consumes accumulated knowledge rather than reconstructing it from documents.

---

# Layered Architecture

```
Knowledge Layer

↓

Reasoning Layer

↓

Decision Layer

↓

Presentation Layer
```

Ownership is strictly separated.

---

# Knowledge Layer

Responsibilities:

* Source ingestion
* Source normalization
* Evidence extraction
* Metadata
* Embeddings
* Incremental construction
* Knowledge retrieval
* Evidence reranking

The Knowledge Layer performs no strategic reasoning.

---

# Knowledge Ontology

```
Source

↓

Evidence

↓

KnowledgeMetadata
```

Supporting entities:

* ExtractionRun
* Embeddings
* Contradictions

---

# Source

Immutable.

Represents an original document.

Owns:

* provenance
* canonical text
* document metadata
* normalization

Never represents strategic knowledge.

---

# Evidence

Immutable.

Represents one reusable strategic or technical claim.

Owns:

* statement
* supporting sources
* evidence type
* extraction provenance

Evidence is the canonical semantic unit.

---

# KnowledgeMetadata

Mutable.

Owns:

* confidence
* credibility
* retrieval priority
* strategic value
* lifecycle state
* quality scores

Evidence never changes.

Metadata evolves.

# Ownership

Knowledge owns:

* Sources
* Evidence
* Metadata
* Embeddings
* ExtractionRuns
* Retrieval

Reasoning owns:

* Engagement
* Decision Model
* Research Object
* Decision Graph

Communication occurs only through the EvidenceRetriever abstraction.

The Knowledge Layer never writes reasoning artifacts.

The Reasoning Layer never accesses storage directly.

# Retrieval Design

Current retrieval is intentionally layered:

1. Lexical retrieval
2. Semantic retrieval
3. Hybrid scoring
4. Candidate generation
5. LLM reranking

The LLM does not generate evidence.

It only selects the best evidence from retrieved candidates.

---

# Design Principles

Always preserve:

1. Ontology before implementation.
2. Stable abstractions.
3. Immutable knowledge.
4. Mutable metadata.
5. Evidence-first reasoning.
6. Retrieval abstraction isolates storage.
7. Incremental knowledge construction.
8. Strategic evidence rather than document transcription.
9. LLMs select evidence—they do not invent evidence.
